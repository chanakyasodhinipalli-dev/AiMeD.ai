## AMIS Architecture (Explain in plain English)

### Layers
1. **Core Runtime**
   - Tenant isolation, config management, runtime policies
2. **Domain Layer**
   - Modules that represent each domain’s workflows and data model
3. **Integration Layer**
   - APIs, events, connectors
4. **Trust Layer**
   - Identity, policy, audit, monitoring
5. **Experience Layer**
   - Dashboards, portals, developer tools, knowledge interfaces

## AMIS vs General-Purpose LLMs (Educational Section)

### General-purpose LLMs
- Great at language and summarization
- Weak at governed, repeatable platform operations
- Not a substitute for system architecture

### AMIS
- A **platform** that can integrate LLM capabilities
- Built for **governance, audit, reliability, and domain workflows**
- Supports “LLM-enabled” experiences without becoming LLM-dependent
